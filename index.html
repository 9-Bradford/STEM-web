Spider
Robot

Autonomous · Wi-Fi · 3D Printed

A fully programmable, four-legged robot — 3D printed from scratch and wirelessly controlled via smartphone using ESP32 and Wi-Fi. Built to transport heavy materials across terrain that wheels cannot handle.

About the ProjectFrame, motors, wiring & code
▾
🖨️ Frame & Body Construction

The entire body and all four legs were 3D printed using PLA filament. Each part was designed in CAD software to fit precise engineering tolerances. The modular design allows individual legs to be replaced without rebuilding the full frame. The body houses the electronics, wiring, and battery pack in a compact, lightweight shell.

⚙️ Servo Motor System

Each of the four legs is powered by multiple servo motors, acting as joints that mimic natural spider movement. The servos receive PWM (Pulse Width Modulation) signals from the ESP32, which precisely control the angle of each joint — enabling smooth, coordinated walking motion including forward, backward, and turning.

📡 ESP32 & Wi-Fi Control

The brain of the robot is an ESP32 microcontroller with built-in Wi-Fi. The robot hosts its own Wi-Fi access point which a smartphone connects to. A custom control interface sends movement commands over the network in real time — no cables or remotes needed, full wireless control from a phone.

💡 Programming & Logic

The ESP32 was programmed using C++ via the Arduino IDE. Custom code handles: Wi-Fi server setup, incoming command parsing, servo timing and synchronisation, and coordinating leg movement in the correct sequence — ensuring all four legs respond accurately and smoothly.

⚡
How It Works5-step build & control guide
▾
01
Design & 3D Print the Frame
All body parts including the chassis, hip joints, and leg segments were designed and 3D printed — rigid enough to carry load yet lightweight enough for the servos to move efficiently.
02
Assemble the Servo Joints
Servo motors are installed at each joint of every leg, carefully calibrated so each servo's centre position aligns with the natural resting stance of the robot.
03
Wire the ESP32 Controller
All servo signal wires are routed to the ESP32's GPIO pins. A power distribution board supplies regulated voltage to the servos separately to prevent resets.
04
Upload the Programme
Custom firmware is uploaded to the ESP32 over USB. The code initialises the Wi-Fi access point, starts the web server, and enters a loop listening for movement commands and driving the legs.
05
Connect & Control via Phone
A phone connects to the robot's Wi-Fi and opens the control interface in a browser. Directional commands are sent over HTTP and the ESP32 moves the legs in real time.

About the ProjectPanels, servos, Arduino & tracking

☀️ Light Detection via Solar Panels

The Solar Flower uses multiple small solar panels on each petal to detect light intensity. When light hits a panel, it generates a measurable voltage — the stronger the light, the higher the voltage. The Arduino reads these values using its ADC pins, determining exactly which direction the brightest light is coming from.

⚙️ Servo Motor Movement

Once the Arduino identifies which panel has the most light, it sends a PWM signal to the servo motors to physically rotate the flower towards that direction — allowing continuous tracking just like real sunflowers through a biological process called heliotropism.

🧠 Arduino Programming Logic

The Arduino code runs a continuous comparison loop: read all solar panel voltages, identify the highest, calculate the difference from the current position, then adjust the servo. A deadzone threshold is built in — small differences are ignored to prevent constant jittering and preserve motor life.

⚡ Energy Efficiency Principle

Solar panels produce the most electricity when facing the light source directly at 90 degrees. Even a 30-degree angle can reduce efficiency by up to 50%. The Solar Flower always maintains the optimal angle — meaning significantly more energy harvested throughout the day compared to a fixed panel.

⚡
How It Works5-step light-tracking process
▾
01
Solar Panels Generate Voltage Readings
Each petal has a solar panel that generates a voltage proportional to the light intensity it receives, fed as analogue signals to the Arduino.
02
Arduino Reads & Compares Values
The Arduino's ADC reads all panel values simultaneously, converting them to digital numbers it can compare — hundreds of times per second.
03
Brightest Direction Identified
The code identifies which panel has the highest voltage. If the difference exceeds the deadzone threshold, a movement command is triggered.
04
Servo Motors Rotate the Flower
A PWM signal is sent to the servo motor, rotating the flower head until all panel voltages are balanced — meaning direct alignment with the light.
05
Continuous Real-Time Tracking
As the light source moves — like the sun across the sky — the flower continuously re-evaluates and adjusts, achieving maximum energy harvest all day automatically.

About the ProjectGlove sensors, servos & ESP32
▾
🧤 The Sensor Glove

The input device is a wearable glove fitted with touch/flex sensors on each finger. When a finger bends, the sensor detects the change in resistance, generating an analogue signal. Each of the five fingers has its own dedicated sensor — meaning the glove can detect independent movement of every finger simultaneously, many times per second.

🦾 The Robotic Hand

The robotic hand is built using servo motors as artificial joints — each finger has servos at the knuckle joints, just like human anatomy. When a servo receives a signal, it rotates to a specific angle, curling or extending the finger. All five fingers move independently and simultaneously, replicating gestures like gripping, pinching, pointing, and waving.

📡 ESP32 Communication

The ESP32 serves as both the input reader and output controller. It reads analogue values from all five sensors, maps them to servo angles, and drives the hand. The loop — read sensor → calculate angle → drive servo — runs fast enough that the hand's movement appears instantaneous and fluid to observers.

💻 Programming Logic

Custom firmware in C++ on the Arduino IDE handles sensor calibration, multi-channel PWM output for all five servos simultaneously, and smooth interpolation to prevent jerky movements. A deadzone filter ignores small unintentional movements so the hand only reacts to deliberate gestures.

⚡
How It Works5-step gesture mirroring process
▾
01
User Wears the Sensor Glove
The operator puts on the glove which has flex/touch sensors on each finger, connecting via wires to the ESP32 control board.
02
Sensors Detect Finger Bends
As each finger bends, its sensor changes resistance — producing a variable voltage the ESP32 reads as an analogue value. Five readings are captured simultaneously, one per finger.
03
ESP32 Processes the Signals
The firmware maps each sensor reading to a servo angle — a fully bent finger maps to 90°, fully straight maps to 0°. Values are filtered to remove noise and prevent jitter.
04
Servo Motors Mirror the Gesture
The ESP32 sends five simultaneous PWM signals to the robotic hand. Each servo rotates to the correct angle, matching the wearer's exact finger position.
05
Real-Time, Fluid Movement
The entire cycle runs at high frequency, creating seamless lag-free mimicry. Move your hand and the robotic hand moves identically, in real time, from across the room.
